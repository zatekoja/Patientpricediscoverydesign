# Documentation Index

This repository uses a simplified documentation structure:

- Active docs: concise, current references for development and operations.
- Archive docs: historical implementation logs, phase notes, and session summaries.

## Active Docs

- Project overview: `README.md`
- Coding agent entrypoint: `AGENTS.md`
- Coding agents navigation guide: `docs/guides/CODING_AGENTS_GUIDE.md`
- Coding standards: `docs/guides/CODING_STANDARDS.md`
- Backend guide: `backend/README.md`
- Backend architecture: `backend/ARCHITECTURE.md`
- Backend quick start: `backend/QUICKSTART.md`
- Observability guide: `OBSERVABILITY.md`
- Backend API notes: `backend/api/README.md`
- Backend workflows notes: `backend/workflows/README.md`
- Backend integration test notes: `backend/tests/integration/README.md`
- Frontend attribution: `Frontend/src/ATTRIBUTIONS.md`
- Frontend guidelines: `Frontend/guidelines/Guidelines.md`

## Archive

Historical documents were moved to:

- `docs/archive/root/`
- `docs/archive/backend/`
- `docs/archive/github/`

Use the archive for project history and deep implementation context from previous phases.

## Documentation Policy

- Put long-lived docs in active locations above.
- Put one-off reports and migration notes in `docs/archive/`.
- Prefer updating an existing active doc over creating a new top-level markdown file.
